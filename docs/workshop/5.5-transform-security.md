## 5.5.1 dbt Full Build | Xây dựng dbt hoàn chỉnh

*[VI: Phần này tải seeds, chạy toàn bộ 17 mô hình dbt và xác minh số lượng hàng trong các bảng mart chính qua Athena.]*

### Set environment variables

dbt reads your bucket path from `S3_DATA_DIR` and `S3_STAGING_DIR`. Set these before running any dbt command:

```bash
export S3_BUCKET_NAME="openaq-pipeline-yourname"   # same value as in Step 5.4
export S3_DATA_DIR="s3://$S3_BUCKET_NAME/processed/"
export S3_STAGING_DIR="s3://$S3_BUCKET_NAME/dbt-staging/"
export AWS_DEFAULT_REGION="ap-southeast-1"
```

> **Why this is required:** `profiles.yml` uses `env_var('S3_DATA_DIR')` to resolve the output path. If the variable is unset, dbt errors with `Env var required but not provided: 'S3_DATA_DIR'`. The CodeBuild runner sets these automatically via Lambda environment variables — they must be set manually for local runs.

### Load seeds

```bash
cd transform/
dbt seed --profiles-dir .
```

Expected output:
```

...
```

Seeds loaded:

|-------|-------------|

```sql
-- AQI correctness: Hanoi 3-year mean PM2.5 should be 35–50 µg/m³
SELECT city,
       ROUND(AVG(avg_value), 1) AS mean_pm25_corrected,
       ROUND(AVG(composite_aqi), 0) AS mean_composite_aqi
FROM openaq_mart.mart_daily_aqi a
JOIN openaq_mart.mart_daily_air_quality q
  ON a.location_id = q.location_id AND a.measurement_date = q.measurement_date
WHERE q.parameter = 'pm25' AND q.is_outlier_station = 0
GROUP BY city;
-- Expected: Hanoi ~40 µg/m³, HCMC ~21 µg/m³
```

```sql
-- Weekend classification correctness (Saturday=6, Sunday=7 in ISO)
SELECT day_type, hour_of_day, ROUND(AVG(avg_value), 1) AS avg_pm25
FROM openaq_mart.mart_diurnal_profile
WHERE parameter = 'pm25' AND city = 'Hanoi'
GROUP BY day_type, hour_of_day
ORDER BY day_type, hour_of_day;
-- Weekend morning peak should be ~1 hour later than weekday (traffic offset)
```

## 5.5.2 Forecast Lambda Container {#forecast-lambda}

The `forecast_generate` Lambda runs as a Docker container image stored in ECR. It was skipped during the first `terraform apply` because the ECR repository existed but had no image.

*[VI: Lambda `forecast_generate` chạy như là Docker container image được lưu trữ trong ECR.]*

### Step 1: Build the Docker image

```bash
cd lambda/forecast_generate/
docker build -t openaq-forecast-generate .
```

Expected build time: ~3–5 minutes (downloads python:3.12-slim, installs statsmodels, pandas, pyarrow, boto3).

### Step 2: Push to ECR

```bash
AWS_ACCOUNT_ID=$(aws sts get-caller-identity --query Account --output text)
AWS_REGION=ap-southeast-1

# Authenticate Docker to ECR
aws ecr get-login-password --region $AWS_REGION \
  | docker login --username AWS \
    --password-stdin $AWS_ACCOUNT_ID.dkr.ecr.$AWS_REGION.amazonaws.com

# Tag and push
docker tag openaq-forecast-generate:latest \
  $AWS_ACCOUNT_ID.dkr.ecr.$AWS_REGION.amazonaws.com/openaq-forecast-generate:latest

docker push \
  $AWS_ACCOUNT_ID.dkr.ecr.$AWS_REGION.amazonaws.com/openaq-forecast-generate:latest
```

### Step 3: Wire the image to Lambda via Terraform

```bash
IMAGE_URI=$AWS_ACCOUNT_ID.dkr.ecr.$AWS_REGION.amazonaws.com/openaq-forecast-generate:latest

cd terraform/
terraform apply -var="forecast_lambda_image_uri=$IMAGE_URI"
```

This second apply modifies one resource (`aws_lambda_function.forecast_generate`) and wires the ECR image. Type `yes`.

### Step 4: Register the external forecast table (one-time)

The forecast Lambda writes Parquet to S3 and expects an Athena external table registered once per environment. Run this DDL in the Athena console (workgroup: `openaq_workgroup`), replacing `{bucket}` with your actual bucket name:

```sql
-- Full DDL is in: transform/setup/create_forecast_table.sql
-- Replace {bucket} with your actual bucket name before running.
-- Note: ${generated_at} and ${model} are Hive partition substitution variables — do not replace these.
CREATE EXTERNAL TABLE IF NOT EXISTS openaq_mart.mart_daily_forecast (
    location_id           INT,
    location_name         STRING,
    city                  STRING,
    forecast_date         DATE,
    forecast_pm25         DOUBLE,
    forecast_aqi          INT,
    forecast_aqi_category STRING,
    ci_lower_95           DOUBLE,
    ci_upper_95           DOUBLE,
    holdout_rmse          DOUBLE
)
PARTITIONED BY (
    generated_at STRING,
    model        STRING
)
STORED AS PARQUET
LOCATION 's3://{bucket}/processed/openaq_mart/mart_daily_forecast/'
TBLPROPERTIES (
    'parquet.compress'                = 'SNAPPY',
    'projection.enabled'              = 'true',
    'projection.generated_at.type'   = 'date',
    'projection.generated_at.range'  = '2026-01-01,NOW',
    'projection.generated_at.format' = 'yyyy-MM-dd',
    'projection.model.type'          = 'enum',
    'projection.model.values'        = 'sarima',
    'storage.location.template'      = 's3://{bucket}/processed/openaq_mart/mart_daily_forecast/generated_at=${generated_at}/model=${model}/'
);
```

### Step 5: Invoke the forecast Lambda

```bash
aws lambda invoke \
  --function-name openaq_forecast_generate \
  --payload '{}' \
  --cli-binary-format raw-in-base64-out \
  /tmp/forecast_response.json

cat /tmp/forecast_response.json
```

Expected response:
```json
{
  "generated_at":   "2026-04-16",
  "stations_ok":    3,
  "errors":         0,
  "sarima_records": 21,
  "alert_count":    0
}
```

> **`stations_ok: 3`** — only 3 of 21 stations have readings within the last 90 days. The remaining 18 stopped reporting to OpenAQ in 2024–2025. The Lambda skips stale stations automatically.

### Verify forecast in Athena

```sql
SELECT location_name, forecast_date, ROUND(forecast_pm25, 1) AS pm25_fcst,
       ROUND(ci_lower_95, 1) AS ci_low, ROUND(ci_upper_95, 1) AS ci_high
FROM openaq_mart.mart_daily_forecast
ORDER BY location_name, forecast_date;
-- Expected: 21 rows (3 stations × 7 days), forecast dates starting tomorrow
```

## 5.5.3 Deploy the Leaflet Dashboard | Triển khai bảng điều khiển Leaflet

The Leaflet station map is served from S3 as a static website. Terraform created the website endpoint and public-read bucket policy during `terraform apply` in Step 5.3. You just need to upload the HTML file with your API URL injected.

*[VI: Bản đồ Leaflet được phục vụ từ S3 dưới dạng trang web tĩnh. Terraform đã tạo website endpoint và bucket policy trong Bước 5.3. Bạn chỉ cần upload file HTML với API URL đã được chèn vào.]*

### Upload the dashboard

```bash
# Inject your API Gateway URL and upload to S3
AQI_API_URL=$(cd terraform && terraform output -raw aqi_api_url)
sed "s|YOUR_API_GATEWAY_URL|$AQI_API_URL|" dashboard/index.html \
  | aws s3 cp - s3://$S3_BUCKET_NAME/dashboard/index.html \
    --content-type text/html
```

### Get the dashboard URL

```bash
terraform -chdir=terraform output -raw dashboard_url
# http://openaq-pipeline-yourname.s3-website-ap-southeast-1.amazonaws.com/dashboard/index.html
```

Open the URL in your browser. You should see 20 colour-coded station markers across Vietnam — 17 over Hanoi (orange, "Unhealthy for Sensitive Groups") and 3 over Ho Chi Minh City (yellow, "Moderate").

> **If the map is blank:** check that the `aqi_api` Lambda executed successfully (`aws logs tail /aws/lambda/openaq_aqi_api`) and that `mart_daily_aqi` has rows (dbt build must have completed first).

> **Note on S3 URL:** The S3 website endpoint serves HTTP only (not HTTPS). The API Gateway URL is HTTPS. Browsers allow mixed content fetch from HTTPS origins to HTTP origins; if you see a CORS or mixed-content error, access the dashboard over HTTP, not HTTPS.

Expected state after a successful load:

- 3 yellow markers over Ho Chi Minh City — "Moderate" (typical PM2.5 ~15–30 µg/m³)

## 5.5.4 Security & IAM Design {#iam} | Bảo mật và Thiết kế IAM

The project applies four security principles throughout:

*[VI: Dự án áp dụng bốn nguyên tắc bảo mật: IAM ít đặc quyền, bí mật trong Secrets Manager, bảo vệ XSS, và giới hạn quét Athena.]*

### Principle 1: Least-privilege IAM per Lambda

A single execution role (`openaq_lambda_role`) is shared, but the **inline policy is resource-scoped** — not wildcards. Each permission covers only the specific ARNs the Lambda actually needs:

| Permission block | Actions | Resource scope |
|-----------------|---------|---------------|
| S3 project bucket | `PutObject`, `GetObject`, `DeleteObject` | `raw/*`, `processed/*`, `athena-results/*` prefixes only |
| S3 OpenAQ archive | `GetObject` | `arn:aws:s3:::openaq-data-archive/records/csv.gz/*` only |
| Glue catalog | `GetDatabase`, `GetTable`, `GetPartitions` | `openaq_raw` and `openaq_mart` databases only |
| Athena | `StartQueryExecution`, `GetQueryExecution`, `GetQueryResults` | `openaq_workgroup` only |
| Kinesis | `PutRecord`, `PutRecords` | `openaq_stream` ARN only |
| Secrets Manager | `GetSecretValue` | `openaq/api_key` ARN only |
| CloudWatch | `PutMetricData` | Namespace `OpenAQ/Pipeline` only |
| SNS | `Publish` | `openaq_alerts` ARN only |

This means a compromised `aqi_api` Lambda cannot write to Kinesis, and a compromised `batch_sync` Lambda cannot call Secrets Manager.

### Principle 2: Secrets Manager — no hardcoded keys

`OPENAQ_API_KEY` is **not** in Lambda environment variables. The streaming producer resolves it at runtime:

```python
# lambda/streaming/handler.py
def _get_api_key() -> str:
    """Fetch API key from Secrets Manager; fall back to env var for local testing."""
    client = boto3.client("secretsmanager", region_name=os.environ["AWS_REGION"])
    secret = client.get_secret_value(SecretId="openaq/api_key")
    return secret["SecretString"]
```

The key is cached in the module-level variable — one Secrets Manager call per Lambda warm container, not per invocation.

### Principle 3: Athena scan guard

The `openaq_workgroup` sets a **10 GB per-query scan limit** via `bytes_scanned_cutoff_per_query`. The workgroup has `enforce_workgroup_configuration = false`, so client-level overrides are permitted, but all dbt and Lambda queries use the workgroup default without overrides. At current data volume (800 MB) this configuration prevents accidental cost spikes from un-partitioned queries while leaving safe headroom.

### Principle 4: XSS protection in the dashboard

Station data from the API (names, categories, values) is injected into the Leaflet popup via `innerHTML`. Without escaping, a malicious station name containing `<script>` tags would execute in the user's browser.

The `escHtml()` function in `dashboard/index.html` sanitises all seven user-visible fields before insertion:

```javascript
function escHtml(s) {
    if (s == null) return "";
    return String(s)
      .replace(/&/g, "&amp;")
      .replace(/</g, "&lt;")
      .replace(/>/g, "&gt;")
      .replace(/"/g, "&quot;")
      .replace(/'/g, "&#39;");
}

// Applied to every field in buildPopup():
const name = escHtml(props.location_name);
const aqi  = escHtml(props.composite_aqi);
// ...
```

### CloudWatch Operational Monitoring

*[VI: Bốn alarm CloudWatch được kích hoạt sau khi triển khai đầy đủ. Alarm ForecastRMSE chỉ tồn tại sau khi Lambda dự báo được kết nối ở Bước 5.5.2.]*

Four alarms are active after full deployment:

| Alarm | Metric | Threshold | Action |
|-------|--------|-----------|--------|
| `openaq_forecast_rmse_sarima_hanoi` | `OpenAQ/Pipeline ForecastRMSE` | > 25 µg/m³ for 3 consecutive days | SNS email to `alert_email` |
| `openaq_missing_stations` | `OpenAQ/Pipeline MissingStations` | > 18 missing stations (fewer than 3 active) for 2 consecutive hours | SNS email to `alert_email` |
| `openaq-kinesis-iterator-age-high` | `AWS/Kinesis GetRecords.IteratorAgeMilliseconds` | > 5 minutes | SNS email to `alert_email` |
| `openaq-dbt-runner-failed` | `AWS/CodeBuild FailedBuilds` | ≥ 1 failed build | SNS email to `alert_email` |

> **`openaq_forecast_rmse_sarima_hanoi`** is created only after the forecast Lambda image is wired in Step 5.5.2. The remaining three alarms are always active after `terraform apply`.

To test an alarm manually:

```bash
aws cloudwatch put-metric-data \
  --namespace "OpenAQ/Pipeline" \
  --metric-name ForecastRMSE \
  --value 30.0 \
  --region ap-southeast-1
```

After 1 such publication (1 evaluation period) you should receive an SNS email alert.
