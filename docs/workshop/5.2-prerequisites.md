# 5.2 Prerequisites | Yêu cầu trước

## IAM Permissions | Quyền IAM

Attach this policy to your `terraform-admin` IAM user. It grants the minimum permissions needed to provision all resources in this workshop.

*[VI: Gắn chính sách này vào người dùng IAM `terraform-admin` của bạn. Nó cấp quyền tối thiểu cần thiết để cung cấp tất cả tài nguyên trong workshop này.]*

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "S3Full",
      "Effect": "Allow",
      "Action": ["s3:*"],
      "Resource": ["arn:aws:s3:::openaq-pipeline-*", "arn:aws:s3:::openaq-pipeline-*/*"]
    },
    {
      "Sid": "GlueFull",
      "Effect": "Allow",
      "Action": ["glue:*"],
      "Resource": ["*"]
    },
    {
      "Sid": "AthenaFull",
      "Effect": "Allow",
      "Action": ["athena:*"],
      "Resource": ["*"]
    },
    {
      "Sid": "KinesisFull",
      "Effect": "Allow",
      "Action": ["kinesis:*", "firehose:*"],
      "Resource": ["*"]
    },
    {
      "Sid": "LambdaAndECR",
      "Effect": "Allow",
      "Action": ["lambda:*", "ecr:*"],
      "Resource": ["*"]
    },
    {
      "Sid": "IAMRolesAndPolicies",
      "Effect": "Allow",
      "Action": ["iam:*"],
      "Resource": [
        "arn:aws:iam::*:role/openaq_*",
        "arn:aws:iam::*:policy/openaq_*",
        "arn:aws:iam::*:role/QuickSightServiceRole-openaq",
        "arn:aws:iam::*:policy/quicksight-openaq-access"
      ]
    },
    {
      "Sid": "QuickSight",
      "Effect": "Allow",
      "Action": ["quicksight:*"],
      "Resource": ["*"]
    },
    {
      "Sid": "CodeBuild",
      "Effect": "Allow",
      "Action": ["codebuild:*"],
      "Resource": ["arn:aws:codebuild:*:*:project/openaq-*"]
    },
    {
      "Sid": "Monitoring",
      "Effect": "Allow",

      "Resource": ["*"]

> **Note on IAM scope:** The Lambda execution role (`openaq_lambda_role`) in this project uses **resource-scoped** policies — not wildcards. For example, the streaming Lambda can only call `kinesis:PutRecord` on the specific stream ARN, and `s3:PutObject` only to the `raw/` and `processed/` prefixes of the project bucket. See [5.5 Transform & Security](5.5-transform-security.md#iam) for the full per-function permission breakdown.

## Required Tools | Công cụ cần thiết

Install and verify each tool before starting:

*[VI: Cài đặt và xác minh từng công cụ trước khi bắt đầu. Tất cả công cụ đều miễn phí và hỗ trợ Windows, macOS và Linux.]*

| Tool | Version | Verify |
|------|---------|--------|
| AWS CLI v2 | Latest | `aws --version` |
| Terraform | ≥ 1.5 | `terraform version` |
| Python | ≥ 3.9 | `python --version` *(Lambda runtime is 3.12; local dbt requires ≥ 3.9)* |
| pip | Latest | `pip --version` |
| Git + Bash | Any | `git --version` |
| Docker Desktop | Latest | `docker version` *(required only for Step 5.5.2 — forecast Lambda image build)* |

### Install AWS CLI (Windows)

*[VI: Tải và chạy trình cài đặt MSI từ trang tài liệu AWS, sau đó cấu hình thông tin đăng nhập.]*

Download and run the MSI installer from the AWS documentation page, then:

```bash
aws configure
# AWS Access Key ID: <your terraform-admin key>
# AWS Secret Access Key: <your terraform-admin secret>
# Default region name: ap-southeast-1
# Default output format: json
```

## Project Setup | Thiết lập Dự án

*[VI: Sao chép kho lưu trữ, tạo file biến Terraform và cài đặt dbt để chuẩn bị cho toàn bộ quá trình triển khai.]*

### 1. Clone the repository

```bash
git clone https://github.com/thanhtrung102/vietnam-air-quality-pipeline
cd vietnam-air-quality-pipeline
```

### 2. Create your Terraform variables file

```bash
cp terraform/terraform.tfvars.example terraform/terraform.tfvars
```

Edit `terraform/terraform.tfvars` and fill in your values:

```hcl
# Must be globally unique — S3 bucket name will be openaq-pipeline-{suffix}
s3_bucket_suffix = "yourname"

aws_region   = "ap-southeast-1"
alert_email  = "you@example.com"

# REQUIRED. Email registered as the QuickSight Author. QuickSight itself is DISABLED in this
# deployment (Standard edition — the quicksight_*.tf are parked in terraform/_qs_disabled/ and
# not loaded), but the variable has no default, so Terraform still requires a value. Any valid
# address works — it provisions nothing while QuickSight is off. Do NOT comment this out.
quicksight_admin_email = "you@example.com"

# NOTE: there is NO openaq_api_key variable. The OpenAQ v3 API key (free at openaq.org/developers)
# is injected into Secrets Manager AFTER apply by terraform/postdeploy.sh (see Step 5.3.3). The
# secret is created with a REPLACE_ME placeholder during apply, then postdeploy.sh overwrites it.
# Lambdas read it from Secrets Manager at runtime — never from env vars or tfvars.
```

> **QuickSight is disabled in this deployment, but `quicksight_admin_email` is still required.** QuickSight needs the **Enterprise** edition; this account is on **Standard**, so all `quicksight_*.tf` are parked in `terraform/_qs_disabled/` (a directory Terraform does not load) and no QuickSight resources are created. However, the `quicksight_admin_email` Terraform variable has **no default**, so `terraform apply` will fail with *"No value for required variable"* if you omit it. Supply any valid email — it provisions nothing while QuickSight is off. The 5.5.4 QuickSight walkthrough is for readers who later upgrade to Enterprise.

> **The OpenAQ API key is NOT a Terraform variable.** Do not add `openaq_api_key` to `terraform.tfvars` — there is no such variable and `terraform apply` would reject it as undeclared. The key is injected into Secrets Manager *after* apply by `terraform/postdeploy.sh` ([Step 5.3.3](5.3-storage-catalog.md)).

> **Do not commit `terraform.tfvars`** — it contains your API key. The `.gitignore` already excludes it.

### 3. Install dbt

```bash
python -m venv .venv
# Activate the virtual environment:
source .venv/bin/activate          # Linux / macOS
source .venv/Scripts/activate      # Windows (Git Bash)
pip install dbt-athena-community==1.10.0
dbt --version
```

### 4. Create your dbt profiles file

dbt reads S3 paths and credentials from `transform/profiles.yml`. This file is gitignored (it contains credentials) — create it from the example:

```bash
cp transform/profiles.yml.example transform/profiles.yml
```

Edit `transform/profiles.yml` and replace the two `YOURNAME` placeholders with your `s3_bucket_suffix` from Step 2, and fill in your `terraform-admin` AWS credentials:

```yaml
s3_staging_dir: "s3://openaq-pipeline-yourname/dbt-staging/"
s3_data_dir:    "s3://openaq-pipeline-yourname/processed/"
...
aws_access_key_id: "AKIA..."
aws_secret_access_key: "..."
```

> **CodeBuild does not need credentials here.** The CodeBuild IAM role (`openaq_dbt_runner_role`) supplies temporary credentials automatically. The key fields are only needed for local `dbt build` runs in Step 5.5.1.

### 5. Verify AWS credentials

```bash
aws sts get-caller-identity
# Should return: Account, UserId, Arn for your terraform-admin user
```

## Expected state after prerequisites | Trạng thái mong đợi

*[VI: Sau khi hoàn thành tất cả các bước trên, hãy xác minh rằng môi trường của bạn đáp ứng các điều kiện sau trước khi chuyển sang Bước 5.3.]*

- `aws sts get-caller-identity` returns `terraform-admin` ARN
- `terraform version` shows ≥ 1.5
- `dbt --version` shows dbt-core 1.x + dbt-athena-community 1.10.0
- `terraform/terraform.tfvars` exists with the four required values: `s3_bucket_suffix`, `aws_region`, `alert_email`, `quicksight_admin_email` (the last is required even though QuickSight is disabled — the variable has no default). **Do not** add `openaq_api_key` — there is no such variable; the key is injected post-apply by `postdeploy.sh`.
- `transform/profiles.yml` exists with `YOURNAME` replaced by your actual bucket suffix
