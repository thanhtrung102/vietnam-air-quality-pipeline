# 5.2 Prerequisites | Yêu cầu trước

## IAM Permissions | Quyền IAM

Attach this policy to your `terraform-admin` IAM user. It grants the minimum permissions needed to provision all resources in this workshop.

*[VI: Gắn chính sách này vào người dùng IAM `terraform-admin` của bạn. Nó cấp quyền tối thiểu cần thiết để cung cấp tất cả tài nguyên trong workshop này.]*

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "S3Full",
      "Effect": "Allow",
      "Action": ["s3:*"],
      "Resource": ["arn:aws:s3:::openaq-pipeline-*", "arn:aws:s3:::openaq-pipeline-*/*"]
    },
    {
      "Sid": "GlueFull",
      "Effect": "Allow",
      "Action": ["glue:*"],
      "Resource": ["*"]
    },
    {
      "Sid": "AthenaFull",
      "Effect": "Allow",
      "Action": ["athena:*"],
      "Resource": ["*"]
    },
    {
      "Sid": "KinesisFull",
      "Effect": "Allow",
      "Action": ["kinesis:*", "firehose:*"],
      "Resource": ["*"]
    },
    {
      "Sid": "LambdaAndECR",
      "Effect": "Allow",
      "Action": ["lambda:*", "ecr:*"],
      "Resource": ["*"]
    },
    {
      "Sid": "IAMRolesAndPolicies",
      "Effect": "Allow",
      "Action": ["iam:*"],
      "Resource": ["arn:aws:iam::*:role/openaq_*", "arn:aws:iam::*:policy/openaq_*"]
    },
    {
      "Sid": "Monitoring",
      "Effect": "Allow",
      "Action": ["logs:*", "cloudwatch:*", "events:*", "scheduler:*", "sns:*", "secretsmanager:*"],
      "Resource": ["*"]
    }
  ]
}
```

> **Note on IAM scope:** The Lambda execution role (`openaq_lambda_role`) in this project uses **resource-scoped** policies — not wildcards. For example, the streaming Lambda can only call `kinesis:PutRecord` on the specific stream ARN, and `s3:PutObject` only to the `raw/` and `processed/` prefixes of the project bucket. See [5.5 Transform & Security](5.5-transform-security.md#iam) for the full per-function permission breakdown.

## Required Tools | Công cụ cần thiết

Install and verify each tool before starting:

| Tool | Version | Verify |
|------|---------|--------|
| AWS CLI v2 | Latest | `aws --version` |
| Terraform | ≥ 1.5 | `terraform version` |
| Python | 3.12+ | `python --version` |
| pip | Latest | `pip --version` |
| Git + Bash | Any | `git --version` |
| Docker Desktop | Latest | `docker version` *(required only for Step 5.5.2 — forecast Lambda image build)* |

### Install AWS CLI (Windows)

Download and run the MSI installer from the AWS documentation page, then:

```bash
aws configure
# AWS Access Key ID: <your terraform-admin key>
# AWS Secret Access Key: <your terraform-admin secret>
# Default region name: ap-southeast-1
# Default output format: json
```

### Install Terraform

Download the binary for your OS from terraform.io, add to PATH, then verify:

```bash
terraform version
# Terraform v1.x.x
```

## Project Setup | Thiết lập Dự án

### 1. Clone the repository

```bash
git clone https://github.com/thanhtrung102/vietnam-air-quality-pipeline
cd vietnam-air-quality-pipeline
```

### 2. Create your Terraform variables file

```bash
cp terraform/terraform.tfvars.example terraform/terraform.tfvars
```

Edit `terraform/terraform.tfvars`:

```hcl
# Must be globally unique — S3 bucket name will be openaq-pipeline-{suffix}
s3_bucket_suffix = "yourname"

aws_region   = "ap-southeast-1"
alert_email  = "you@example.com"

# Your OpenAQ v3 API key (free at openaq.org/developers)
# This is only used during terraform apply to seed Secrets Manager.
# The Lambda reads it from Secrets Manager at runtime — not from env vars.
openaq_api_key = "your-openaq-api-key"
```

> **Do not commit `terraform.tfvars`** — it contains your API key. The `.gitignore` already excludes it.

### 3. Install dbt

```bash
python -m venv .venv
source .venv/Scripts/activate   # Windows: .venv\Scripts\activate
pip install dbt-athena-community==1.10.0
dbt --version
```

### 4. Verify AWS credentials

```bash
aws sts get-caller-identity
# Should return: Account, UserId, Arn for your terraform-admin user
```
