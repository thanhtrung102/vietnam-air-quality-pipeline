# 5.6 Cleanup | Dọn dẹp tài nguyên

Remove all AWS resources created by this workshop to stop incurring charges. Follow the steps in order — some steps must precede others (e.g. S3 must be emptied before Terraform can delete the bucket).

*[VI: Xóa tất cả tài nguyên AWS được tạo bởi workshop này để ngừng tính phí. Thực hiện các bước theo thứ tự.]*

---

## Step 1: Stop Scheduled Triggers | Dừng lịch trình tự động

Stop EventBridge Scheduler rules first so no Lambda invocations run while you are cleaning up.

```bash
# List all OpenAQ schedules
aws scheduler list-schedules \
  --query "Schedules[?contains(Name, 'openaq')].Name" \
  --output text

# Disable each schedule (replace names if yours differ)
aws scheduler update-schedule \
  --name openaq_batch_sync_trigger \
  --state DISABLED \
  --flexible-time-window Mode=OFF \
  --schedule-expression "rate(1 day)" \
  --target '{"Arn": "placeholder", "RoleArn": "placeholder"}'
```

> **Easier alternative:** In the AWS console → EventBridge → Schedules → select all `openaq_*` schedules → Actions → Disable.

---

## Step 2: Empty S3 Buckets | Xóa nội dung S3

Terraform cannot delete an S3 bucket that contains objects. Empty it first:

```bash
export S3_BUCKET_NAME="openaq-pipeline-yourname"

aws s3 rm s3://$S3_BUCKET_NAME/ --recursive
```

> **Warning:** This permanently deletes all ingested data, processed Parquet files, and Athena query results. Confirm you do not need any of this data before running.

Verify the bucket is empty:

```bash
aws s3 ls s3://$S3_BUCKET_NAME/ --recursive | wc -l
# Expected: 0
```

---

## Step 3: Destroy All Terraform Resources | Phá hủy tài nguyên Terraform

```bash
cd terraform/
terraform destroy
```

Type `yes` when prompted. Terraform will destroy all provisioned resources in reverse dependency order.

Expected output on completion:

```
Destroy complete! Resources: 40 destroyed.
```

Resources destroyed include: S3 bucket, Glue databases and tables, Athena workgroup, IAM role and policy, Kinesis stream, Firehose delivery stream, all Lambda functions, EventBridge schedules, SNS topic, Secrets Manager secret, CloudWatch alarms.

> **Secrets Manager deletion delay:** By default `recovery_window_in_days = 0` is set in `terraform/secrets.tf`, enabling immediate deletion. If you see a Secrets Manager deletion error, wait 10 seconds and re-run `terraform destroy`.

---

## Step 4: Delete ECR Repository | Xóa ECR Repository

The ECR repository and forecast image are not managed by Terraform (to avoid accidentally destroying the image during routine applies). Delete it manually:

```bash
aws ecr delete-repository \
  --repository-name openaq-forecast-generate \
  --force \
  --region ap-southeast-1
```

The `--force` flag deletes all images in the repository before removing it.

---

## Step 5: Drop Athena External Table | Xóa bảng Athena bên ngoài

The `mart_daily_forecast` external table was registered manually in [Step 5.5.2](5.5-transform-security.md#forecast-lambda) and is not managed by Terraform. Drop it in the Athena console:

```sql
DROP TABLE IF EXISTS openaq_mart.mart_daily_forecast;
```

Run in the Athena console with workgroup `openaq_workgroup` (before Terraform destroy removes the workgroup), or run it now if `terraform destroy` has already completed — either Athena workgroup will work.

---

## Step 6: Delete CloudWatch Log Groups | Xóa nhóm log CloudWatch

Terraform does not delete CloudWatch Log Groups that contain retained logs. Delete them manually:

```bash
for fn in batch_sync streaming_producer weather_ingest aqi_api forecast_generate completeness_check; do
  aws logs delete-log-group \
    --log-group-name /aws/lambda/openaq_${fn} \
    --region ap-southeast-1
done
```

---

## Step 7: Verify Billing | Kiểm tra thanh toán

Check the AWS Billing dashboard to confirm charges have stopped:

1. AWS Console → Billing → Bills → Current month
2. Filter by service — all `openaq-*` tagged resources should show $0 after the cleanup date
3. Check **Cost Explorer** for any lingering charges (note: billing dashboard updates within 24 hours)

Common charges that may still appear temporarily:
- S3 DELETE requests (~$0.005 per 1,000)
- Lambda invocations that ran before you disabled EventBridge (a few cents at most)
- Athena queries that completed before cleanup
