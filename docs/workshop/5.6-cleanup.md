# 5.6 Cleanup | Dọn dẹp tài nguyên

Remove all AWS resources created by this workshop to stop incurring charges. Follow the steps in order — some steps must precede others (e.g. S3 must be emptied before Terraform can delete the bucket).

*[VI: Xóa tất cả tài nguyên AWS được tạo bởi workshop này để ngừng tính phí. Thực hiện các bước theo thứ tự.]*

---

## Step 1: Stop Scheduled Triggers | Dừng lịch trình tự động

Stop EventBridge Scheduler rules first so no Lambda invocations run while you are cleaning up.

*[VI: Dừng các quy tắc EventBridge Scheduler trước để không có lần gọi Lambda nào chạy trong khi bạn đang dọn dẹp.]*

```bash
# List all OpenAQ schedules (expect: openaq_batch_daily, openaq_streaming_30min,
# openaq_completeness_hourly, openaq_weather_daily, openaq_dbt_daily, openaq_forecast_daily)
aws scheduler list-schedules \
  --query "Schedules[?contains(Name, 'openaq')].Name" \
  --output text
```

> **Easiest approach:** In the AWS console → EventBridge → Schedules → select all `openaq_*` schedules → Actions → Disable. This avoids having to re-supply `--schedule-expression` and `--target` for each schedule.

---

## Step 2: Empty S3 Buckets | Xóa nội dung S3

Terraform cannot delete an S3 bucket that contains objects. Empty it first:

*[VI: Terraform không thể xóa S3 bucket chứa các đối tượng. Hãy làm trống bucket trước khi chạy destroy.]*

```bash
export S3_BUCKET_NAME="openaq-pipeline-yourname"

aws s3 rm s3://$S3_BUCKET_NAME/ --recursive
```

> **Warning:** This permanently deletes all ingested data, processed Parquet files, and Athena query results. Confirm you do not need any of this data before running.

Verify the bucket is empty:

```bash
aws s3 ls s3://$S3_BUCKET_NAME/ --recursive | wc -l
# Expected: 0
```

---

## Step 3: Delete ECR Images | Xóa image ECR

The ECR repository (`openaq-forecast-generate`) is managed by Terraform, but the Terraform resource does not set `force_delete = true`. This means `terraform destroy` will fail if the repository contains images. Delete the images first:

*[VI: ECR repository được quản lý bởi Terraform nhưng không có `force_delete`. Xóa image trước khi chạy `terraform destroy` để tránh lỗi.]*

```bash
aws ecr delete-repository \
  --repository-name openaq-forecast-generate \
  --force \
  --region ap-southeast-1
```

The `--force` flag deletes all images in the repository before removing it. Terraform will then have nothing to delete and will skip the ECR resource cleanly.

---

## Step 4: Drop Athena External Table | Xóa bảng Athena bên ngoài

The `mart_daily_forecast` external table was registered manually in [Step 5.5.2](5.5-transform-security.md#forecast-lambda) and is not managed by Terraform. Drop it before running `terraform destroy` — the `openaq_workgroup` must still exist to execute this DDL.

*[VI: Bảng `mart_daily_forecast` được đăng ký thủ công ở Bước 5.5.2 và không do Terraform quản lý. Xóa bảng này **trước** khi chạy `terraform destroy` vì bước tiếp theo sẽ xóa workgroup Athena.]*

```sql
DROP TABLE IF EXISTS openaq_mart.mart_daily_forecast;
```

Run this in the Athena console (workgroup: `openaq_workgroup`).

---

## Step 5: Destroy All Terraform Resources | Phá hủy tài nguyên Terraform

*[VI: Chạy `terraform destroy` để xóa tất cả ~93 tài nguyên AWS theo thứ tự phụ thuộc ngược lại. Gõ `yes` khi được nhắc.]*

```bash
cd terraform/
terraform destroy
```

Type `yes` when prompted. Terraform will destroy all provisioned resources in reverse dependency order.

Expected output on completion:

```
Destroy complete! Resources: ~93 destroyed.
```

Resources destroyed include: S3 bucket, Glue databases and tables, Athena workgroup, IAM roles and policies, Kinesis stream, Firehose delivery stream, all Lambda functions, EventBridge schedules, SNS topics, Secrets Manager secret, CloudWatch alarms, ECR repository (if images were already deleted), CodeBuild project.

> **Secrets Manager deletion delay:** `terraform/secrets.tf` sets `recovery_window_in_days = 7`. Terraform will schedule the secret for deletion, but it will only be permanently deleted after 7 days. If you need immediate irrecoverable deletion (e.g. billing demo cleanup), run:
> ```bash
> aws secretsmanager delete-secret \
>   --secret-id openaq/api_key \
>   --force-delete-without-recovery
> ```
> before (or instead of) `terraform destroy`.

---

## Step 6: Delete CloudWatch Log Groups | Xóa nhóm log CloudWatch

*[VI: Terraform không xóa các CloudWatch Log Group chứa log đã lưu giữ. Xóa thủ công bằng vòng lặp shell dưới đây.]*

Terraform does not delete CloudWatch Log Groups that contain retained logs. Delete them manually:

```bash
for fn in batch_sync streaming_producer weather_ingest aqi_api forecast_generate completeness_check; do
  aws logs delete-log-group \
    --log-group-name /aws/lambda/openaq_${fn} \
    --region ap-southeast-1
done
```

---

## Step 7: Delete QuickSight Resources | Xóa tài nguyên QuickSight

QuickSight resources (dashboard, template, analysis, datasets, data source) are managed by Terraform and will be destroyed in Step 5. However, QuickSight **Enterprise Edition** incurs a monthly Author seat charge (~$18/month) that continues until you cancel the subscription. Cancel it separately after `terraform destroy`.

*[VI: Tài nguyên QuickSight được Terraform quản lý và sẽ bị xóa trong Bước 5. Tuy nhiên, **Enterprise Edition** tính phí Author ~$18/tháng cho đến khi bạn hủy đăng ký. Hủy riêng sau khi `terraform destroy`.]*

```bash
# Verify Terraform destroyed all QuickSight resources
aws quicksight list-dashboards --aws-account-id $(aws sts get-caller-identity --query Account --output text) \
  --region ap-southeast-1 --query 'DashboardSummaryList[*].Name'
# Expected: [] (empty list — dashboard deleted by terraform destroy)
```

Then cancel the Enterprise subscription to stop the Author seat charge:

1. QuickSight console → top-right username → **Manage QuickSight**
2. **Account settings** → **Terminate account**
3. Type `confirm` → **Delete account**

> **Note:** Terminating QuickSight deletes all dashboards, analyses, and datasets in the account. Only do this after `terraform destroy` has run.

---

## Step 8: Verify Billing | Kiểm tra thanh toán

*[VI: Kiểm tra bảng điều khiển thanh toán AWS để xác nhận đã ngừng tính phí. Lưu ý: bảng điều khiển thanh toán cập nhật trong vòng 24 giờ.]*

Check the AWS Billing dashboard to confirm charges have stopped:

1. AWS Console → Billing → Bills → Current month
2. Filter by service — all `openaq-*` tagged resources should show $0 after the cleanup date
3. Check **Cost Explorer** for any lingering charges (note: billing dashboard updates within 24 hours)

Common charges that may still appear temporarily:
- S3 DELETE requests (~$0.005 per 1,000)
- Lambda invocations that ran before you disabled EventBridge (a few cents at most)
- Athena queries that completed before cleanup
- QuickSight Author seat charge if subscription was not cancelled in Step 7
