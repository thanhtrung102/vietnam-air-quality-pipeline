# 5.1 Introduction | Giới thiệu

## What You Will Build | Bạn sẽ xây dựng gì

In this workshop you will build a fully serverless air quality analytics pipeline on AWS. By the end you will have:

- **A live Leaflet station map** showing real-time AQI for 21 Vietnamese monitoring stations, coloured by US EPA health category
- **A SARIMA forecast Lambda** producing daily 7-day PM2.5 predictions for active stations with CloudWatch RMSE monitoring
- **Fully reproducible infrastructure** — every AWS resource is declared in Terraform; the entire environment can be torn down and rebuilt with two commands

*[VI: Trong workshop này bạn sẽ xây dựng một pipeline phân tích chất lượng không khí hoàn toàn serverless trên AWS. Cuối cùng bạn sẽ có bản đồ Leaflet trực tiếp, Lambda dự báo SARIMA và hạ tầng có thể tái tạo đầy đủ.]*

## Architecture Overview | Tổng quan Kiến trúc

*[VI: Pipeline có ba đường nạp dữ liệu song song hội tụ tại Athena, sau đó qua dbt và Lambda API đến bản đồ Leaflet trực tiếp.]*

> Architecture diagram: [docs/architecture.drawio](../architecture.drawio) — open in [diagrams.net](https://app.diagrams.net) to view the full interactive diagram.

The pipeline has three parallel ingestion paths that all converge at Athena:

```
OpenAQ S3 Archive (CSV.GZ)
  └─ batch_sync Lambda (daily) ──────────────────────── raw/batch/
                                                               │
OpenAQ REST API v3                                            │
  └─ streaming_producer Lambda (every 30 min)                 │
       └─ Kinesis → Firehose ─────────────────────── raw/stream/
                                                               │
Open-Meteo ERA5 Reanalysis                                    │
  └─ weather_ingest Lambda (daily) ──────────────── raw/weather/
                                                               │
                         Glue Data Catalog (partition projection)
                                                               │
                                                          Athena
                                                               │
                                                    dbt (17 models)
                                                               │
                                                      aqi_api Lambda
                                                               │
                                                  API Gateway → Leaflet Map
```

**Why serverless?**
No persistent compute means ~$1.61/month in production. There is no EMR cluster to manage, no Redshift warehouse to provision, and no broker to operate. All compute scales automatically from zero.

*[VI: Tại sao serverless? Không có tính toán liên tục nghĩa là khoảng $1.61/tháng trong sản xuất. Không có cụm EMR nào cần quản lý.]*

## AWS Services Used | Dịch vụ AWS sử dụng

| Service | Role in this workshop |
|---------|----------------------|
| **Amazon S3** | Data lake — raw and processed Parquet prefixes; static website host for Leaflet map |
| **AWS Glue Data Catalog** | Schema registry with partition projection — queries are instant, no Crawlers needed |
| **Amazon Athena** | SQL engine over S3 Parquet; dbt writes mart tables here |
| **Amazon Kinesis Data Streams** | Buffers real-time OpenAQ API records (ON_DEMAND capacity) |
| **Amazon Kinesis Firehose** | Delivers Kinesis records as GZIP NDJSON to S3 every 5 minutes |
| **AWS Lambda** | 6 functions: `batch_sync`, `streaming_producer`, `weather_ingest`, `aqi_api`, `forecast_generate`, `completeness_check` |
| **Amazon EventBridge Scheduler** | Six schedules: `batch_sync` 01:00 UTC · `weather_ingest` 02:00 UTC · dbt (CodeBuild) 02:30 UTC · `forecast_generate` 03:00 UTC; `streaming_producer` every 30 min; `completeness_check` every hour |
| **Amazon SNS** | Email alerts from CloudWatch alarms (`ForecastRMSE`, `MissingStations`) |
| **AWS Secrets Manager** | Stores `OPENAQ_API_KEY` — never hardcoded in environment variables |
| **Amazon CloudWatch** | Custom metrics `ForecastRMSE` and `ActiveStations`; alarms fire when RMSE > 25 µg/m³ or station coverage drops below 85% |
| **Amazon ECR** | Hosts the Docker container image for the `forecast_generate` Lambda |

*[VI: Mỗi dịch vụ AWS trong bảng trên đóng một vai trò cụ thể trong pipeline — từ lưu trữ thô đến truy vấn phân tích, từ streaming đến dashboard trực tiếp và cảnh báo vận hành.]*

## Learning Objectives | Mục tiêu học tập

After completing this workshop you will be able to:

1. Configure **Glue partition projection** to make S3 data queryable in Athena instantly, without running a Crawler
2. Write **Terraform modules** that provision a multi-service AWS pipeline from a single `terraform apply`
3. Build a **dbt transformation layer** on Athena, including AQI computation with US EPA 2024 breakpoints
4. Package a **Python ML model** (SARIMA) as a Docker container Lambda and deploy it from ECR
5. Apply **least-privilege IAM** — per-function roles with resource-scoped action grants
6. Protect a public dashboard against **XSS injection** and store secrets securely in Secrets Manager

*[VI: Sau khi hoàn thành workshop này bạn sẽ có thể: cấu hình Glue partition projection, viết Terraform modules, xây dựng dbt transformation layer, đóng gói model ML dưới dạng container Lambda, áp dụng IAM ít đặc quyền nhất, và bảo vệ dashboard công khai khỏi XSS injection.]*

## Data Description | Mô tả dữ liệu

*[VI: Pipeline sử dụng hai nguồn dữ liệu mở: OpenAQ cho các phép đo chất lượng không khí và Open-Meteo ERA5 cho dữ liệu khí tượng tái phân tích.]*

### Source: OpenAQ

OpenAQ is an open platform aggregating air quality data worldwide. This workshop uses:

- **21 Vietnamese monitoring stations** — 17 in Hanoi, 4 in Ho Chi Minh City
- **18 reference-grade instruments** (FEM MONRE government stations) + **3 AirGradient low-cost sensors**
- **Parameters in scope:** PM2.5 and PM10 (AQI computed); NO₂, O₃, CO, SO₂ retained raw but excluded from AQI (sub-daily averaging windows not available at daily grain)
- **Coverage:** January 2023 – present; ~900,000 hourly readings

**Known data quality issues:**
- Sentinel value -999.0 (~3.8% of rows) — filtered at staging
- Station 6273386 (VNUHCMUS Campus 1): artefact PM2.5 readings up to 2,000 µg/m³ — marked `is_outlier_station = 1`, excluded from city-level aggregations
- Low-cost sensor overread: AirGradient PMS5003 (3 stations) may read high in humid conditions; `sensor_type` flag is retained for downstream filtering. No flat correction is applied — ERA5 relative-humidity-based correction is noted as future work in `int_measurements_enriched.sql`

### Source: Open-Meteo ERA5

ERA5 is ECMWF's fifth-generation atmospheric reanalysis. Open-Meteo provides free, no-key API access covering:
- Temperature at 2 m, relative humidity at 2 m
- Wind speed and direction at 10 m
- Precipitation, surface pressure, atmospheric boundary layer height (BLH)

BLH is used to derive the `inversion_risk` flag (BLH < 500 m AND wind < 2 m/s) — a key predictor of PM2.5 episodes.

## Cost Estimate | Ước tính chi phí

| Service | Monthly estimate |
|---------|----------------|
| Amazon S3 (50 GB) | ~$1.15 |
| Amazon Athena (50 queries × 63.6 KB avg scan) | ~$0.00 |
| AWS Lambda (6 functions × daily invocations) | ~$0.00 (Free Tier) |
| Amazon Kinesis ON_DEMAND | ~$0.01 |
| Amazon CloudWatch (2 alarms) | ~$0.30 |
| Amazon ECR (1.5 GB image) | ~$0.15 |
| **Total** | **~$1.61/month** |

The key cost saving is **partition projection** — Athena scans an average of 63.6 KB per query rather than the full ~800 MB table. At $5/TB, this makes analytical queries effectively free.

*[VI: Tiết kiệm chi phí chính là partition projection — Athena quét trung bình 63.6 KB mỗi truy vấn thay vì toàn bộ bảng ~800 MB.]*
