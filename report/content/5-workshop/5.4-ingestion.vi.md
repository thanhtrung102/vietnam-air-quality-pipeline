Ba hàm Lambda (python3.12 / arm64), được điều khiển bởi EventBridge Scheduler, đưa dữ liệu thô vào S3.

## Các hàm
| Function | Mem / Timeout | Schedule | Source → Sink |
|---|---|---|---|
| `batch_sync` | 512 MB / 900 s | daily 01:00 UTC | OpenAQ S3 archive → `raw/batch/` (DLQ wired) |
| `streaming_producer` | 256 MB / 120 s | every 30 min | OpenAQ REST API v3 → Kinesis → Firehose → `raw/stream/` (DLQ wired) |
| `weather_ingest` | 256 MB / 900 s | daily 02:00 UTC | Open-Meteo ERA5 → `raw/weather/` |

- **Luồng streaming:** `streaming_producer` → **Kinesis Data Streams** (ON_DEMAND, KMS) → **Firehose**
  (GZIP, buffer 128 MB / 300 s) → `raw/stream/yyyy/MM/dd/HH/`.
- OpenAQ key được đọc từ **Secrets Manager** (`openaq/api_key`) — không bao giờ từ biến môi trường dạng plaintext.
- EventBridge Scheduler gọi mỗi Lambda thông qua **execution role** của nó (không phải resource policy).

## Xác minh
```bash
aws lambda invoke --function-name openaq_batch_sync /tmp/out.json && cat /tmp/out.json
aws scheduler list-schedules --query 'Schedules[?starts_with(Name,`openaq`)].Name'
aws kinesis describe-stream-summary --stream-name openaq_stream --query 'StreamDescriptionSummary.StreamStatus'
```
Một lần chạy thành công sẽ báo cáo số lượng thành công theo từng trạm; các lỗi xuất hiện dưới dạng metric `BatchStationFailures`
trên CloudWatch (và các DLQ ghi nhận các lỗi bất đồng bộ).
