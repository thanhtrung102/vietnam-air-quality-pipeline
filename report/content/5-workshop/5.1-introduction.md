**Objective —** understand the end-to-end shape of the pipeline (the three ingestion paths, the
dbt-on-Athena transform, and the serving layer) before you build it section by section.

The pipeline has **three parallel ingestion paths** that converge at Athena, are transformed by dbt, and
served to a live map + analytics dashboard, with a SARIMA forecaster on the side.

```
OpenAQ S3 archive ──(batch_sync, daily)─────────────────────────┐
OpenAQ REST API v3 ─(streaming_producer, 30 min)→ Kinesis→Firehose┤→ S3 raw/
Open-Meteo ERA5 ───(weather_ingest, daily)──────────────────────┘    │
                                            Glue catalog (partition projection)
                                                          │
                       dbt-on-Athena (CodeBuild, daily) → 13 of 17 marts (Parquet)
                                                          │
                    ┌──────────────────────┬─────────────────────────┐
              aqi_api + API GW        forecast_generate          completeness_check
                    │                  (SARIMA 7-day)            (hourly monitor)
              Leaflet + Chart.js dashboard (S3 static site)      CloudWatch + SNS
```

**Data:** 21 OpenAQ stations (17 Hanoi-area, 4 HCMC) + Open-Meteo ERA5 weather. AQI uses the **US EPA
2024** PM2.5/PM10 breakpoints; gases (NO₂/O₃/SO₂/CO) are retained raw but excluded from the AQI (they
need ppm/ppb + sub-daily windows — documented as a non-goal at daily grain).

**Verified live (2026-06-02), end-to-end:** `streaming_producer` → 66 readings ok / 0 failed;
`forecast_generate` → 35 SARIMA records / 0 errors; **dbt CodeBuild `PASS=84 WARN=0 ERROR=0 TOTAL=84`**;
API `GET /` → 200 (5 active stations); `/analytics/forecast` → 35 rows; all `/analytics/*` → 200;
AWS Budget live.
