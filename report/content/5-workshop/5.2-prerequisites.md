## Tooling
- **AWS account** + an admin-capable IAM user; AWS CLI configured for `ap-southeast-1`.
- **Terraform ≥ 1.10** (uses the native S3 state lockfile — no DynamoDB).
- **Python 3.12** (to build the Lambda zips), **bash**, **git**.
- An **OpenAQ API key** (free) and an email for SNS alerts.

## Configuration
Clone the repo and create `terraform/terraform.tfvars` from the example:

```hcl
s3_bucket_suffix   = "yourinitials"          # bucket = openaq-pipeline-<suffix>
alert_email        = "you@example.com"
monthly_budget_usd = 8                        # AWS Budget ceiling
# forecast_lambda_image_uri = "<ecr-uri>"     # leave unset for a fresh clone (forecast gated off)
```

The OpenAQ API key is **not** a Terraform variable — it is injected into Secrets Manager post-deploy
(`postdeploy.sh`), keeping it out of state.

## Build the Lambda zips (required before the first apply)
```bash
bash lambda/build.sh        # packages all Lambda deployment zips
```

> Skipping this step makes the first `terraform apply` fail with a `filebase64sha256` error — the zips
> must exist before apply.
