**Objective —** resolve the issues most likely to bite during a fresh build. Each entry is a real problem
hit while deploying and verifying this stack live, with the fix.

## `terraform apply` fails with `filebase64sha256: no such file`
The Lambda zips don't exist yet. Terraform hashes each deployment zip at plan time.

```bash
bash lambda/build.sh        # then re-run terraform apply
```

Re-run `build.sh` after any change to Lambda source — `terraform apply` re-packages and redeploys from the
freshly built zips.

## An Athena query is rejected for scanning too much data
The `openaq_workgroup` enforces a **10 GB per-query scan cap** as a cost guardrail. A query that would
scan more is cancelled by design. Add a partition predicate (`WHERE year = … AND month = …`) so partition
projection prunes the scan, or raise the cap in `terraform/athena.tf` if you genuinely need it.

## `/analytics/forecast` returns an empty `rows` array
Expected on a fresh clone — the forecast Lambda is `count`-gated on `forecast_lambda_image_uri` and is
skipped until you provide the image. Build it via CodeBuild `openaq-forecast-image`, set the URI in
`terraform.tfvars`, and re-apply. See **5.5 → ML**.

## Editing dbt models has no effect on the marts
Pushing dbt changes to git does not deploy them. The CodeBuild source is `codebuild-source.zip`, which is
re-packaged by `terraform apply`. Run `terraform apply`, then start a build:

```bash
aws codebuild start-build --project-name openaq-dbt-runner
```

## `terraform plan` always shows 2 resources to add after a clean apply
{{% notice note %}}
This is the **SNS email subscription** (`aws_sns_topic_subscription`). Until you click the confirmation
link AWS emails you, the subscription is `PendingConfirmation` and Terraform cannot reconcile it — so it
re-proposes it on every plan. Confirm the subscription from your inbox and the perpetual diff disappears.
It is harmless either way; alarms still fire to confirmed subscribers.
{{% /notice %}}

## `terraform plan` crashes locally ("plugin did not respond")
The AWS provider plugin can crash when the **temp drive is full** — it writes large state/cache files to
`%TEMP%`. On Windows, point temp at a drive with headroom before planning:

```powershell
$env:TMP = "D:\Temp"; $env:TEMP = "D:\Temp"
terraform plan
```

## A brand-new station or date has no rows
Partition projection (no crawler) only exposes partitions inside the **projection range** declared in the
Glue table. If you ingest data outside the configured `year`/`month` range, widen the projection bounds in
`terraform/glue.tf` and re-apply — there is no crawler to auto-discover it.

## The dashboard map is empty
The map reads `GET /` (last 7 days, latest reading per station). If no station has reported in 7 days the
map is empty by design — check that `streaming_producer` is running (5.4) and that `mart_daily_aqi` has
recent rows. The HCMC stations are decommissioned; their history lives in the **Analytics** tab.
