**Objective —** wire the three ingestion Lambdas (batch, streaming, weather) to EventBridge Scheduler and
confirm raw data is landing in S3, with the OpenAQ key sourced from Secrets Manager.

Three Lambda functions (python3.12 / arm64), driven by EventBridge Scheduler, land raw data in S3.

## Functions
| Function | Mem / Timeout | Schedule | Source → Sink |
|---|---|---|---|
| `batch_sync` | 512 MB / 900 s | daily 01:00 UTC | OpenAQ S3 archive → `raw/batch/` (DLQ wired) |
| `streaming_producer` | 256 MB / 120 s | every 30 min | OpenAQ REST API v3 → Kinesis → Firehose → `raw/stream/` (DLQ wired) |
| `weather_ingest` | 256 MB / 900 s | daily 02:00 UTC | Open-Meteo ERA5 → `raw/weather/` |

- **Streaming path:** `streaming_producer` → **Kinesis Data Streams** (ON_DEMAND, KMS) → **Firehose**
  (GZIP, 128 MB / 300 s buffer) → `raw/stream/yyyy/MM/dd/HH/`.
- The OpenAQ key is read from **Secrets Manager** (`openaq/api_key`) — never from plaintext env.
- EventBridge Scheduler invokes each Lambda via its **execution role** (not a resource policy).

## Verify
```bash
aws lambda invoke --function-name openaq_streaming_producer /tmp/out.json && cat /tmp/out.json
aws scheduler list-schedules --query 'Schedules[?starts_with(Name,`openaq`)].Name'
aws kinesis describe-stream-summary --stream-name openaq_stream --query 'StreamDescriptionSummary.StreamStatus'
```

{{% notice info %}}
**Expected output.** The invoke returns a per-station summary — on the live deployment
`streaming_producer` reported **66 readings ok / 0 failed**; `list-schedules` returns **6** `openaq…`
schedules; and the stream status is `"ACTIVE"`. Failures surface as the `BatchStationFailures` CloudWatch
metric, and the DLQs capture async failures.
{{% /notice %}}
