**Mục tiêu —** chuyển đổi data lake thô thành các mart đã được kiểm thử bằng dbt-on-Athena, expose chúng
qua Lambda `aqi_api` + API Gateway, render dashboard trực tiếp, và thêm dự báo SARIMA 7 ngày.

## Chuyển đổi — dbt-on-Athena (CodeBuild)
Dự án CodeBuild `openaq-dbt-runner` chạy dbt hàng ngày (02:30 UTC). dbt-athena-community build
**17 model** (2 staging + 2 intermediate + 13 marts); bản build mặc định là **13 trong 17**
(`--exclude tag:bi_disabled` bỏ qua 4 mart chẩn đoán/dư thừa).

- **Logic AQI:** ngưỡng (breakpoint) PM2.5/PM10 của US EPA 2024, nội suy tuyến tính từng đoạn; AQI tổng hợp =
  chất ô nhiễm tệ nhất (PM2.5 thắng khi hòa).
- **Tests (84, tất cả pass):** generic (not_null / relationships / accepted_values / dbt_utils range) +
  4 singular (freshness ×2, bất biến composite-AQI, bất biến health-category) + **2 unit test**
  (phép tính EPA breakpoint + composite/tie-break, đầu vào được mock ~0 scan) + **dbt-expectations**
  (row-count, `max≥avg≥min`, tính không âm). Xem `docs/DATA-QUALITY.md`.

{{% notice note %}}
**Lưu ý khi triển khai.** Chỉnh sửa các tệp dbt và push lên git **không** triển khai chúng —
`terraform apply` đóng gói lại `codebuild-source.zip`, và bản build theo lịch (hoặc thủ công) tiếp theo
sẽ nhận chúng.
{{% /notice %}}

```bash
aws codebuild start-build --project-name openaq-dbt-runner
```

{{% notice info %}}
**Kết quả mong đợi.** Bản build chạy ~7 phút và kết thúc `SUCCEEDED`. Tóm tắt dbt test ở cuối log build
hiển thị **`PASS=84 WARN=0 ERROR=0 SKIP=0 TOTAL=84`** — toàn bộ 84 test đều xanh (dbt-athena chạy một
truy vấn Athena cho mỗi test, nên toàn bộ lần chạy vẫn nằm trong ngân sách scan).
{{% /notice %}}

## Phục vụ — API + dashboard
- Lambda `aqi_api` đứng sau API Gateway: `GET /` → GeoJSON trên `mart_daily_aqi` (7 ngày gần nhất, mới nhất
  theo từng trạm); `GET /analytics/{health,seasonal,compliance,forecast}` → JSON cho dashboard.
- **Dashboard** = `dashboard/index.html` (Leaflet + Chart.js) trên S3 static site; Terraform inject
  URL API trực tiếp. Tab Map + tab Analytics (4 sheet).

```bash
curl -s "$API/" | python -c "import sys,json;print(len(json.load(sys.stdin)['features']),'stations')"
curl -s "$API/analytics/forecast" | python -c "import sys,json;print(len(json.load(sys.stdin)['rows']),'rows')"
```

{{% notice info %}}
**Kết quả mong đợi.** `GET /` trả về **`5 stations`** (tập các trạm đang báo cáo) và `/analytics/forecast`
trả về **`35 rows`** (5 trạm × chân trời 7 ngày). Cả bốn endpoint `/analytics/*` đều trả về HTTP 200.
{{% /notice %}}

## ML — dự báo SARIMA (container Lambda)
`forecast_generate` (ECR image) khớp mô hình SARIMA(1,1,1)(1,0,1,7) cho mỗi trạm đang hoạt động, ghi
`mart_daily_forecast` (35 dòng / 5 trạm / 7 ngày), và phát holdout-RMSE lên CloudWatch.

{{% notice tip %}}
**Dự báo bị tắt (gated off) trên bản clone mới.** Lambda dự báo được `count`-gated trên
`forecast_lambda_image_uri`, nên lần apply đầu tiên bỏ qua nó (đó là lý do plan hiển thị ~82 tài nguyên,
chưa phải toàn bộ). Để bật: build image qua CodeBuild `openaq-forecast-image`, đặt URI thu được vào
`terraform.tfvars`, và apply lại. Endpoint `/analytics/forecast` trả về mảng `rows` rỗng cho đến lúc đó —
dashboard xử lý trường hợp này một cách mượt mà.
{{% /notice %}}

## Bảo mật
- **Secrets Manager** cho OpenAQ key (không có biến môi trường plaintext); IAM theo nguyên tắc đặc quyền tối thiểu (role của dbt runner được
  giới hạn phạm vi vào `openaq_mart*`); API Gateway throttle (burst 20 / rate 10) + reserved concurrency 10;
  truy cập công khai S3 được giới hạn phạm vi vào `dashboard/*`. WAF là một quyết định từ chối có lý do, được biện minh trong phạm vi (envelope).
