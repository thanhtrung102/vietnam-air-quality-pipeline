**Mục tiêu —** cài đặt công cụ, thiết lập thông tin xác thực và `terraform.tfvars`, và build các tệp zip
Lambda để lần `terraform apply` đầu tiên thành công.

## Công cụ
- **Tài khoản AWS** + một người dùng IAM có quyền admin; AWS CLI được cấu hình cho `ap-southeast-1`.
- **Terraform ≥ 1.10** (sử dụng S3 state lockfile gốc — không dùng DynamoDB).
- **Python 3.12** (để build các tệp zip Lambda), **bash**, **git**.
- Một **OpenAQ API key** (miễn phí) và một email để nhận cảnh báo SNS.

## Cấu hình
Clone repo và tạo `terraform/terraform.tfvars` từ tệp ví dụ:

```hcl
s3_bucket_suffix   = "yourinitials"          # bucket = openaq-pipeline-<suffix>
alert_email        = "you@example.com"
monthly_budget_usd = 8                        # AWS Budget ceiling
# forecast_lambda_image_uri = "<ecr-uri>"     # leave unset for a fresh clone (forecast gated off)
```

OpenAQ API key **không** phải là một biến Terraform — nó được inject vào Secrets Manager sau khi triển khai
(`postdeploy.sh`), giúp giữ nó nằm ngoài state.

## Build các tệp zip Lambda (bắt buộc trước lần apply đầu tiên)
```bash
bash lambda/build.sh        # packages all Lambda deployment zips
```

{{% notice warning %}}
**Build các tệp zip trước lần apply đầu tiên.** Bỏ qua `lambda/build.sh` sẽ khiến `terraform apply` thất
bại với lỗi `filebase64sha256: open … no such file` — Terraform tính hash từng tệp zip tại thời điểm plan,
nên các tệp phải tồn tại trước. Chạy lại `build.sh` mỗi khi bạn thay đổi mã nguồn Lambda.
{{% /notice %}}
