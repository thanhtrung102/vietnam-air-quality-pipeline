**Objective —** tear the stack down cleanly and confirm the state is empty, so costs return to ~$0.

Everything is Terraform-managed, so teardown is a single command.

```bash
cd terraform
terraform destroy        # removes all ~82 resources
```

## Notes
- **S3 buckets** with versioning may need objects emptied first if `force_destroy` is not set:
  ```bash
  aws s3 rm s3://openaq-pipeline-<suffix> --recursive
  ```
- The **remote state bucket** (`openaq-tfstate-thanhtrung102`) is bootstrapped out-of-band and is *not*
  destroyed by `terraform destroy` — remove it manually only if you are decommissioning entirely.
- **ECR image** (`openaq-forecast-generate`) and the **CodeBuild** source object persist if created
  outside the main apply; delete them manually if desired.
- Cost after teardown returns to ~$0; the storage/catalog/state shells cost cents until removed.

## Verify teardown
```bash
aws lambda list-functions --query 'Functions[?starts_with(FunctionName,`openaq`)].FunctionName'
terraform plan
```

{{% notice info %}}
**Expected output.** The first command returns an empty array `[]` (no `openaq…` functions remain), and
`terraform plan` proposes re-creating everything from scratch — both confirm the destroy was complete and
the state is clean.
{{% /notice %}}
