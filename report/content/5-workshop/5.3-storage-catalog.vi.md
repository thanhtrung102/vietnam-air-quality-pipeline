Lớp này cung cấp S3 data lake, Glue catalog (partition projection — không dùng crawler), và
Athena workgroup. Nó được tạo bởi cùng một lệnh `terraform apply` với phần còn lại của stack.

## Triển khai
```bash
cd terraform
terraform init        # configures the remote S3 backend (versioned, SSE, native lockfile)
terraform plan        # review — a fresh clone shows ~82 resources to add (forecast gated off)
terraform apply
```

## Những gì được tạo ra
- **S3 bucket** `openaq-pipeline-<suffix>` — bật versioning, AES256 + bucket-key SSE, static website
  (`dashboard/*` công khai), lifecycle: `athena-results/` hết hạn 7 ngày, `raw/stream/` 60 ngày, bản noncurrent 7 ngày,
  `processed/` → Intelligent-Tiering.
- **Glue databases** `openaq_raw` + `openaq_mart`; 3 bảng raw external (`batch`, `stream`, `weather`)
  với **partition projection** — không có crawler nào chạy, nên không phát sinh chi phí crawl theo mỗi lần scan.
- **Athena workgroup** `openaq_workgroup` — kết quả ghi vào `s3://…/athena-results/`, SSE_S3, **giới hạn scan 10 GB
  cho mỗi truy vấn** (rào chắn chi phí).

## Xác minh
```bash
aws glue get-databases --query 'DatabaseList[?starts_with(Name,`openaq`)].Name'
aws athena get-work-group --work-group openaq_workgroup --query 'WorkGroup.State'   # ENABLED
```
