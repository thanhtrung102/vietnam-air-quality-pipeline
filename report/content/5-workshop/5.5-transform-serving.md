**Objective —** transform the raw lake into tested marts with dbt-on-Athena, expose them through the
`aqi_api` Lambda + API Gateway, render the live dashboard, and add the SARIMA 7-day forecast.

## Transform — dbt-on-Athena (CodeBuild)
The `openaq-dbt-runner` CodeBuild project runs dbt daily (02:30 UTC). dbt-athena-community builds
**17 models** (2 staging + 2 intermediate + 13 marts); the default build is **13 of 17**
(`--exclude tag:bi_disabled` skips 4 diagnostic/redundant marts).

- **AQI logic:** US EPA 2024 PM2.5/PM10 breakpoints, piecewise-linear interpolation; composite AQI =
  worst pollutant (PM2.5 wins ties).
- **Tests (84, all pass):** generic (not_null / relationships / accepted_values / dbt_utils range) +
  4 singular (freshness ×2, composite-AQI invariant, health-category invariant) + **2 unit tests**
  (EPA breakpoint math + composite/tie-break, mocked inputs ~0 scan) + **dbt-expectations**
  (row-count, `max≥avg≥min`, non-negativity). See `docs/DATA-QUALITY.md`.

{{% notice note %}}
**Deploy note.** Editing dbt files and pushing to git does **not** deploy them — `terraform apply`
re-packages `codebuild-source.zip`, and the next scheduled (or manual) build picks it up.
{{% /notice %}}

```bash
aws codebuild start-build --project-name openaq-dbt-runner
```

{{% notice info %}}
**Expected output.** The build runs ~7 minutes and finishes `SUCCEEDED`. The dbt test summary at the tail
of the build log reads **`PASS=84 WARN=0 ERROR=0 SKIP=0 TOTAL=84`** — all 84 tests green (dbt-athena runs
one Athena query per test, so the run stays inside the scan budget).
{{% /notice %}}

## Serving — API + dashboard
- `aqi_api` Lambda behind API Gateway: `GET /` → GeoJSON over `mart_daily_aqi` (last 7 days, latest
  per station); `GET /analytics/{health,seasonal,compliance,forecast}` → JSON for the dashboard.
- **Dashboard** = `dashboard/index.html` (Leaflet + Chart.js) on the S3 static site; Terraform injects
  the live API URL. Map tab + Analytics tab (4 sheets).

```bash
curl -s "$API/" | python -c "import sys,json;print(len(json.load(sys.stdin)['features']),'stations')"
curl -s "$API/analytics/forecast" | python -c "import sys,json;print(len(json.load(sys.stdin)['rows']),'rows')"
```

{{% notice info %}}
**Expected output.** `GET /` returns **`5 stations`** (the actively-reporting set) and
`/analytics/forecast` returns **`35 rows`** (5 stations × 7-day horizon). All four `/analytics/*`
endpoints return HTTP 200.
{{% /notice %}}

## ML — SARIMA forecast (container Lambda)
`forecast_generate` (ECR image) fits SARIMA(1,1,1)(1,0,1,7) per active station, writes
`mart_daily_forecast` (35 rows / 5 stations / 7-day), and emits holdout-RMSE to CloudWatch.

{{% notice tip %}}
**Forecast is gated off on a fresh clone.** The forecast Lambda is `count`-gated on
`forecast_lambda_image_uri`, so the first apply skips it (that is why the plan shows ~82 resources, not
the full set). To enable it: build the image via CodeBuild `openaq-forecast-image`, set the resulting URI
in `terraform.tfvars`, and re-apply. The `/analytics/forecast` endpoint returns an empty `rows` array
until then — the dashboard handles this gracefully.
{{% /notice %}}

## Security
- **Secrets Manager** for the OpenAQ key (no plaintext env); least-privilege IAM (the dbt runner role is
  scoped to `openaq_mart*`); API Gateway throttle (burst 20 / rate 10) + reserved concurrency 10;
  S3 public access scoped to `dashboard/*`. WAF is a documented, envelope-justified decline.
