# Feature Lifecycle

Large or correctness-critical work gets its own folder here instead of living in
`process/general-plans/`. Copy `_feature-template/` to `process/features/{feature-slug}/` and use:

| Folder | Holds |
|---|---|
| `active/` | the in-progress `YYYY-MM-DD-{slug}_PLAN.md` |
| `completed/` | archived plans (searchable decision history) |
| `backlog/` | deferred ideas — check before duplicating |
| `reports/` | execution / validation outputs + evidence packs |
| `references/` | research that informs the work |

Cross-cutting plans that don't warrant a feature folder stay in `process/general-plans/`.
See `process/context/planning/all-planning.md` for naming and depth conventions.
